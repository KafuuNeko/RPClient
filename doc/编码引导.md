# AI 编码引导

> 本文档是项目快速导航。凡是项目结构、核心流程、入口文件、依赖注册、数据模型、页面路由或重要开发规范发生变化，都必须同步更新本文档。若本文档与代码冲突，以代码为准，并在本次变更中修正文档。

## 0. 开工前先读

1. 先读本文档，确定要改的业务域和入口文件。
2. 涉及 MVI、Compose、状态流转时，再读 `doc/MVI框架开发规范指南.md`。
3. 涉及 Prompt、世界书、总结机制时，再读：
   - `doc/SillyTavern Prompts 结构与世界书触发机制说明.md`
   - `doc/SillyTavern 总结机制说明.md`
4. 修改代码后，如果新增或改变了本文件记录的入口、流程、约束、目录职责，必须同时更新本文档。

## 1. 项目概览

- 项目类型：Android 单 module 应用。
- 语言与 UI：Kotlin + Jetpack Compose。
- 架构：自定义 MVI，核心基类在 `app/src/main/java/me/kafuuneko/rpclient/libs/core/`。
- 依赖注入：Koin，注册入口在 `app/src/main/java/me/kafuuneko/rpclient/RPClientApp.kt`。
- 持久化：Room + Kotpref。
- 网络：OkHttp + Gson。
- 主要业务：角色卡、世界书、聊天会话、Prompt 构建、LLM Provider 管理、请求日志。

常用命令：

```powershell
.\gradlew.bat :app:assembleDebug
.\gradlew.bat :app:testDebugUnitTest
```

主要版本集中在 `gradle/libs.versions.toml`，不要分散写死依赖版本。

## 2. 顶层目录

| 路径 | 职责 |

| `gradle/libs.versions.toml` | Gradle 版本目录 |

## 3. 应用启动与全局注册

| 文件 | 作用 |
| --- | --- |
| `app/src/main/AndroidManifest.xml` | Activity 注册、启动页、权限 |
| `app/src/main/java/me/kafuuneko/rpclient/RPClientApp.kt` | Application、Kotpref 初始化、Koin 模块注册、Room 创建 |

新增全局服务、Repository、Builder、Factory 时，优先检查是否需要在 `RPClientApp.kt` 的 `appModules` 中用 `singleOf(::Xxx)` 或 `single { ... }` 注册。

## 4. MVI 基础设施

| 文件 | 重点 |
| --- | --- |
| `libs/core/CoreActivity.kt` | Compose `setContent`、主题包裹、Edge-to-edge |
| `libs/core/CoreActivityWithEvent.kt` | 收集 ViewEvent，处理 Toast、跳转、setResult |
| `libs/core/CoreViewModel.kt` | UiIntent 分发、`@UiIntentObserver` 反射绑定、UiState、异步任务队列 |
| `libs/core/CoreViewModelWithEvent.kt` | 一次性 ViewEvent、`AppViewEvent` |

约定：

- ViewModel 继承 `CoreViewModelWithEvent<UiIntent, UiState>` 居多。
- Intent 处理函数用 `@UiIntentObserver(XxxUiIntent.SomeIntent::class)` 标注。
- 状态更新使用 `copy(...)` 后调用 `setup()`。
- 页面跳转优先用 `AppViewEvent.StartActivity(...)`。
- 需要等待 Activity 消费完事件时用 `emitAndAwait()`，常见于创建成功后跳转并关闭当前页。

更完整规范见 `doc/MVI框架开发规范指南.md`。

## 5. Feature 模块地图

每个 feature 通常包含：

- `<Feature>Activity.kt`：收集 `uiStateFlow`，把状态和 `emitIntent` 传给 Layout。
- `<Feature>ViewModel.kt`：处理业务逻辑、Repository 调用、状态转换、ViewEvent。
- `presentation/`：`UiState`、`UiIntent`、可选 `ViewEvent`。
- `model/`：当前 feature 专用 UI model 或表单。
- `ui/`：Compose 布局。

| Feature | 入口 | 主要职责 | 常关联文件 |
| --- | --- | --- | --- |
| `main` | `MainActivity.kt` / `MainViewModel.kt` | 首页、最近会话、设置页入口、全局设置 | `MainUiState.kt`、`MainUiIntent.kt`、`MainLayout.kt` |
| `chat` | `ChatActivity.kt` / `ChatViewModel.kt` | 实际聊天、消息发送、流式生成、重生成、总结、会话设定 | `ChatPromptBuilder.kt`、`SummaryPromptBuilder.kt`、`ChatRepository.kt` |
| `chatcreate` | `ChatCreateActivity.kt` / `ChatCreateViewModel.kt` | 新建聊天、选择角色、选择开场白、绑定世界书条目 | `ChatCreateForm.kt`、`PromptMacroResolver.kt` |
| `characterlist` | `CharacterListActivity.kt` / `CharacterListViewModel.kt` | 角色列表、进入新增或编辑 | `CharacterRepository.kt` |
| `characteredit` | `CharacterEditActivity.kt` / `CharacterEditViewModel.kt` | 创建/编辑角色、头像选择、角色字段维护 | `CharacterEditForm.kt`、`FileRepository.kt` |
| `worldbooklist` | `WorldBookListActivity.kt` / `WorldBookListViewModel.kt` | 世界书列表、进入新增或编辑 | `LorebookRepository.kt` |
| `worldbookedit` | `WorldBookEditActivity.kt` / `WorldBookEditViewModel.kt` | 世界书信息与条目列表 | `WorldBookEditForm.kt`、`WorldBookEntryEditActivity.kt` |
| `worldbookentryedit` | `WorldBookEntryEditActivity.kt` / `WorldBookEntryEditViewModel.kt` | 世界书条目新增/编辑，关键字、深度、排序等 | `WorldBookEntryEditForm.kt` |
| `llmproviderlist` | `LLMProviderListActivity.kt` / `LLMProviderListViewModel.kt` | LLM Provider 列表、启停、进入编辑 | `LLMRepository.kt` |
| `llmprovideredit` | `LLMProviderEditActivity.kt` / `LLMProviderEditViewModel.kt` | Provider 新增/编辑、测试连接、协议配置 | `LLMProviderEditForm.kt`、`LLMClientFactory.kt` |
| `promptpreset` | `PromptPresetActivity.kt` / `PromptPresetViewModel.kt` | 编辑主 Prompt 与总结 Prompt | `AppModel.kt`、`PromptType.kt` |
| `requestlog` | `RequestLogActivity.kt` / `RequestLogViewModel.kt` | 查看和清空 LLM 请求日志 | `LLMRequestLogRepository.kt` |
| `about` | `AboutActivity.kt` | 纯展示页面 | `AboutLayout.kt`、`AppLibs.kt` |

定位规则：

- 改页面交互：先找对应 `feature/<name>/presentation/*UiIntent.kt` 和 `*ViewModel.kt`。
- 改页面显示数据：先找 `*UiState.kt`、`model/`、`ui/*Layout.kt`。
- 改跨页面业务能力：优先找 `libs/room/repository/`、`libs/prompt/`、`libs/llm/`。
- 改 Activity 参数：同步修改跳转方、目标 Activity 的 `EXTRA_*`、`AndroidManifest.xml` 如需注册新页。

## 6. 页面跳转与参数

| 目标页面 | 参数 |
| --- | --- |
| `ChatActivity` | `EXTRA_SESSION_ID`，字符串形式 session id |
| `CharacterEditActivity` | `EXTRA_CHARACTER_ID`，Long，缺省或 0 表示新增 |
| `WorldBookEditActivity` | `EXTRA_LOREBOOK_ID`，Long，缺省或 0 表示新增 |
| `WorldBookEntryEditActivity` | `EXTRA_LOREBOOK_ID` 必需，`EXTRA_ENTRY_ID` 可选 |
| `LLMProviderEditActivity` | `EXTRA_PROVIDER_ID`，Long，缺省或 0 表示新增 |

跳转通常由 ViewModel 发出 `AppViewEvent.StartActivity`，Activity 只负责消费事件。不要在 Compose 里直接启动 Activity。

## 7. 数据层地图

Room 入口：

- `libs/room/AppDatabase.kt`：数据库实体、版本、DAO 抽象方法。
- `libs/room/Converters.kt`：Room 类型转换。
- `libs/room/MutableDao.kt`：通用 insert/update/delete。
- `libs/room/entity/`：Room 实体。
- `libs/room/dao/`：SQL 查询。
- `libs/room/repository/`：业务读写封装，ViewModel 通常只依赖 Repository。

实体与 Repository：

| 业务 | Entity | DAO | Repository |
| --- | --- | --- | --- |
| 角色 | `Character.kt` | `CharacterDao.kt` | `CharacterRepository.kt` |
| 世界书 | `Lorebook.kt`、`LorebookEntry.kt` | `LorebookDao.kt`、`LorebookEntryDao.kt` | `LorebookRepository.kt` |
| 聊天 | `ChatSession.kt`、`ChatMessage.kt` | `ChatSessionDao.kt`、`ChatMessageDao.kt` | `ChatRepository.kt` |
| LLM Provider | `LLMProvider.kt` | `LLMProviderDao.kt` | `LLMRepository.kt` |
| 请求日志 | `LLMRequestLog.kt` | `LLMRequestLogDao.kt` | `LLMRequestLogRepository.kt` |
| 文件 | `FileEntity.kt` | `FileDao.kt` | `FileRepository.kt` |

注意：

- `AppDatabase` 当前版本为 `1`，`exportSchema = false`。
- 现在 Room Builder 使用了 `allowMainThreadQueries()`，但新代码仍应把 IO 操作放到 `Dispatchers.IO`。
- 新增实体或字段时，需要同时更新 Entity、DAO、Repository、`AppDatabase`，并评估数据库版本和迁移策略。

## 8. LLM 调用链

核心文件：

| 文件 | 作用 |
| --- | --- |
| `libs/llm/LLMClient.kt` | Provider 适配器统一接口：非流式 `generate`、流式 `streamGenerate` |
| `libs/llm/LLMClientFactory.kt` | 根据 `LLMProviderProtocol` 创建具体适配器 |
| `libs/llm/model/LLMModels.kt` | 通用 Provider、Message、Request、Response、StreamEvent 模型 |
| `libs/llm/adapter/OpenAICompatibleLLMClient.kt` | OpenAI 兼容协议 |
| `libs/llm/adapter/GeminiLLMClient.kt` | Gemini 协议 |
| `libs/llm/adapter/AnthropicMessagesLLMClient.kt` | Anthropic Messages 协议 |
| `libs/llm/adapter/LLMHttpUtils.kt` | HTTP 请求辅助与日志相关模型 |
| `libs/room/repository/LLMRepository.kt` | Provider 管理、默认 Provider 初始化、调用 Factory |

新增 Provider 协议时，通常需要改：

1. `LLMProviderProtocol`。
2. 新增 `libs/llm/adapter/*LLMClient.kt`。
3. `LLMClientFactory.create(...)`。
4. `LLMRepository.defaultProviders()`，如果要预置模板。
5. `llmprovideredit` 的表单与 UI，如果协议配置项有变化。

## 9. Prompt 与总结链路

核心文件：

| 文件 | 作用 |
| --- | --- |
| `libs/prompt/PromptBuildContext.kt` | 构建 prompt 需要的上下文聚合 |
| `libs/prompt/ChatPromptBuilder.kt` | 组织主 Prompt、角色设定、总结、世界书、历史、后置指令 |
| `libs/prompt/SummaryPromptBuilder.kt` | 构建总结请求，并选择本次总结要标记的消息 |
| `libs/prompt/PromptMacroResolver.kt` | 解析 `{{user}}`、`{{char}}`、`{{history}}` 等宏 |
| `libs/prompt/FormattedHistoryBuilder.kt` | 聊天历史格式化 |
| `libs/prompt/WorldBookActivator.kt` | 根据上下文触发世界书条目 |
| `libs/AppModel.kt` | 默认主 Prompt、默认总结 Prompt、世界书预算、总结配置 |

聊天生成主路径：

1. `ChatViewModel.onSendMessage()` 保存用户消息。
2. `ChatViewModel.buildGenerationRequest()` 聚合 session、character、messages、worldbook entries、provider。
3. `ChatPromptBuilder.build(...)` 生成 `LLMGenerationRequest`。
4. `LLMRepository.generateWithSelectedProvider(...)` 或 `streamGenerateWithSelectedProvider(...)` 调用适配器。
5. 结果写回 `ChatRepository.createMessage(...)` 或流式更新后 `updateMessageContent(...)`。

自动总结主路径：

1. `ChatViewModel.maybeAutoSummarize(...)` 检查 `AppModel.autoSummaryEnabled`。
2. `ChatViewModel.summarizeSession(...)` 读取未总结消息和当前 Provider。
3. `SummaryPromptBuilder.build(...)` 生成请求。
4. `SummaryPromptBuilder.selectSummarizedMessageIds(...)` 决定哪些消息可标记为已总结。
5. `ChatRepository.saveSessionSummarize(...)` 保存 summary 并标记消息。

改 Prompt 或总结逻辑前，先读 `doc/SillyTavern Prompts 结构与世界书触发机制说明.md` 和 `doc/SillyTavern 总结机制说明.md`。

## 10. UI 与资源约定

- 全局主题在 `ui/theme/`。
- 通用组件在 `ui/widgets/`，例如 `AppTopBar.kt`、`RpCommonViews.kt`。
- 页面 UI 在各自 `feature/<name>/ui/`。
- 字符串统一写入 `app/src/main/res/values/strings.xml`，不要在 UI 中散落可复用文案。
- Compose 层只渲染 `UiState` 并发出 `UiIntent`，不要直接访问 Repository、Room、Kotpref 或 LLM。

## 13. 变更后的同步检查

每次代码更新后，检查是否需要同步本文档：

- 新增、删除、重命名 feature。
- 新增 Activity、跳转参数或 `EXTRA_*`。
- 新增全局服务、Repository、Builder、Factory 或 Koin 注册。
- Room Entity、DAO、Repository、数据库版本变化。
- LLM 协议、Provider 类型、默认 Provider、请求/响应模型变化。
- Prompt 构建、宏、世界书触发、总结策略变化。
- MVI 基类或开发规范变化。
- 重要命令、依赖、Gradle module 结构变化。

如果更新了相关设计文档，也在本文档的引用列表中同步补充。
